### Command Permissions

Configure which Discord roles can use specific commands:

```yaml
permissions:
  ranking_commands:
    required_roles: ["Admin", "Moderator"]
    blacklisted_roles: ["Muted", "Restricted"]
    
  information_commands:
    required_roles: ["Member"]
    channel_restrictions: ["#bot-commands", "#staff-only"]
    
  management_commands:
    required_roles: ["Owner", "Co-Owner"]
    require_confirmation: true
```

## Event Handling

Synera responds to various Discord events to maintain synchronization:

### Member Events

<Steps>
  <Step title="Member Join">
    When a new member joins the Discord server, Synera automatically checks their Roblox account and assigns appropriate roles
  </Step>
  <Step title="Member Leave">
    Logs member departures and optionally removes them from Roblox group if configured
  </Step>
  <Step title="Role Changes">
    Monitors Discord role changes and synchronizes with Roblox ranks when necessary
  </Step>
  <Step title="Nickname Updates">
    Tracks nickname changes and can enforce Roblox username synchronization
  </Step>
</Steps>

### Server Events

- **Guild Updates**: Responds to server setting changes
- **Channel Events**: Adapts to channel creation/deletion
- **Permission Changes**: Updates command availability based on role permissions
